### 2. Heading Structure — `_layouts/*.html` ✅
**Instances: 448 | Priority: High**

Several layouts use `<h3>` as the first/primary heading on the page. Screen readers
and WCAG require headings to be logically nested starting from `<h1>`.

- [x] Audit each layout file in `_layouts/` for the first heading element used
- [x] Change primary page headings from `<h3>` / `<h2>` to `<h1>` where appropriate
- [x] Ensure heading levels increment without skipping (h1 → h2 → h3, not h1 → h3)

Fixed in: `introlinux.html`, `introdata.html`, `introinfosec.html`, `introit.html`, `intropython.html`, `advlinux.html`, `AdvSec.html`, `AdvData.html`, `advdata.html`, `AdvDataOverview.html`, `TopicTemplate.html`
- Page titles: `<h2>` → `<h1>`
- Sub-sections (Learning Objectives, Activities): `<h5>` → `<h2>`
- TopicTemplate: `<h3>` → `<h1>`

pages written as plain HTML before the Jekyll setup was added.

- [x] Find all non-Jekyll HTML files: `grep -rL "layout:" --include="*.html" .`
- [x] Add `lang="en"` to the `<html>` tag in each
- [x] Add a descriptive `<title>` in the `<head>` of each

**Already resolved:** All non-Jekyll HTML files are lab pages generated by `tex-to-html.py`,
which already outputs `<html lang="en-US">` and a `<title>` in every file. Jekyll pages
get `<html lang="en">` from `_layouts/default.html`. No files were actually missing these.

### 6. Skip Link Target Missing — `_layouts/default.html` ✅
**Instances: 35 | Priority: Medium**

`default.html` includes `<a href="#main-content" class="skip-link">Skip to main content</a>`
but no element in the layout has `id="main-content"`, so the skip link goes nowhere.

- [x] Added `<main id="main-content">` wrapping `{{ content }}` in `default.html`
- [x] Removed orphaned `</main>` that was misplaced after the footer
- [x] Course layouts already had `id="main-content"` on their content div

### 7. Duplicate Section IDs (`#one`, `#two`) ✅
**Instances: 22 | Priority: Low**

`stylesheet.css` defines styles for `#one`, `#two`, `#three` as background-colour
variants for paragraph sections. These IDs are reused across many pages, making each
page invalid HTML (IDs must be unique within a document).

- [x] Converted `id="one"`, `id="two"`, `id="three"` to `class="one"`, `class="two"`, `class="three"` in all 9 affected pages
- [x] Updated `stylesheet.css` selectors from `#one`/`#two`/`#three` to `.one`/`.two`/`.three`
- [x] Visual appearance unchanged
