### 2. Heading Structure — `_layouts/*.html` ✅
**Instances: 448 | Priority: High**

Several layouts use `<h3>` as the first/primary heading on the page. Screen readers
and WCAG require headings to be logically nested starting from `<h1>`.

- [x] Audit each layout file in `_layouts/` for the first heading element used
- [x] Change primary page headings from `<h3>` / `<h2>` to `<h1>` where appropriate
- [x] Ensure heading levels increment without skipping (h1 → h2 → h3, not h1 → h3)

Fixed in: `introlinux.html`, `introdata.html`, `introinfosec.html`, `introit.html`, `intropython.html`, `advlinux.html`, `AdvSec.html`, `AdvData.html`, `advdata.html`, `AdvDataOverview.html`, `TopicTemplate.html`
- Page titles: `<h2>` → `<h1>`
- Sub-sections (Learning Objectives, Activities): `<h5>` → `<h2>`
- TopicTemplate: `<h3>` → `<h1>`

### 11. NaN:1 Contrast — Navbar Dropdowns and Skip Link ✅
**Instances: ~500 | Priority: High**

pa11y (Puppeteer) reports `NaN:1` when it can't compute contrast because an element's
`background-color` is `transparent` and the ancestor background is set via an inline
style (which Puppeteer doesn't reliably traverse).

- [x] Moved navbar background from inline `style="background-color: #2c365e"` on `<nav>`
  to `nav.navbar { background-color: #2c365e !important; }` in `stylesheet.css`
- [x] Changed `.skip-link` from `background: #000` (shorthand) to `background-color: #000000`
  (explicit property) and `color: #fff` → `color: #ffffff` (fully explicit values)
- [x] Same explicit values applied to `.skip-link:focus`
- [ ] Re-scan to confirm NaN resolves *(fixes applied, re-scan not yet run)*
